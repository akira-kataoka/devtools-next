Phase 578 — REST API 401 エラー時の診断ガイド強化 🆕

[A] BEFORE — generic な案内

⚠ HTTP 401 (SF GET) INVALID_SESSION_ID Session expired or invalid
💡 セッションの有効期限が切れています。Salesforce へログインし直し、ポップアップの ⟳ ボタンで再接続してください (Lightning ドメインの sid は REST から使えない場合があります)。
🔧 セッション設定を開く → 📜 ログイン履歴を確認する →

⚠ 「Lightning sid 疑いです」が文中に埋もれてて見落とす。再接続ボタンは別画面に行かないと押せない

[B] AFTER 🆕 — Lightning sid 検出時の親切な診断

⚠ HTTP 401 (SF GET) INVALID_SESSION_ID Session expired or invalid
💡 ⚠ 現在 Lightning ドメイン (.lightning.force.com) の sid を使用中。これは REST API で 401 になります。my.salesforce.com タブを一度開いてから ⟳ 再接続してください。
🌐 my.salesforce.com を新タブで開く → ⟳ 再接続を実行 → 🔧 セッション設定を開く → 📜 ログイン履歴を確認する →

✓ 原因 (Lightning sid) を診断 + 1-クリックで my.salesforce.com 開く + 1-クリックで再接続

[C] AFTER 🆕 — 通常の sid 期限切れ (Lightning ではない)

⚠ HTTP 401 (SF GET) INVALID_SESSION_ID Session expired or invalid
💡 sid 取得元: .my.salesforce.com。セッション有効期限切れの可能性があります。my.salesforce.com を開き直してから ⟳ 再接続してください。
🌐 my.salesforce.com を新タブで開く → ⟳ 再接続を実行 → 🔧 セッション設定を開く → 📜 ログイン履歴を確認する →

✓ Lightning ではない場合でも cookieDomain を表示 + 1-クリック修復

Phase 578 で変更: 401 エラー時に state.sessionCookieDomain を診断材料として使い、Lightning ドメインの sid を掴んでいる時は明示的に警告。新アクション 2 種 openMySalesforce (新タブで my.salesforce.com を開いて API 用 cookie をセット) と reconnect (⟳ 再接続を 1 click 発火) を追加。ユーザー報告「REST API がエラーだけ返る (401)」への直接対処。
